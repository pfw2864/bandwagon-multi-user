## 搬瓦工配置多用户：Linux 系统账号管理完整步骤

### 第一种场景：给朋友或同事建一个普通 SSH 登录账号

这是最常见的需求。新建一个账号，朋友用自己的密码登进来，能在自己的 home 目录里干活，但碰不到系统级别的东西。

**1. 创建新用户**

bash
adduser 用户名


系统会提示你设置密码，以及填一些个人信息（全回车跳过就行）。用 `adduser` 而不是 `useradd` 的原因是它会自动创建 `/home/用户名` 目录，省事。

**2. 验证账号已建好**

bash
id 用户名


能看到 uid/gid 信息就说明建成了。

**3. 让朋友用新账号登录**

他们用同样的 SSH 命令，把用户名换成刚建的那个，端口和 IP 一样：

bash
ssh 用户名@你的IP -p 你的端口号


登进去之后是普通权限，`/root` 目录进不去，系统配置文件改不了，各自玩各自的。

---

### 第二种场景：给账号加 sudo 权限（管理员级别）

如果要给某个人管理员级别的操作权限，就需要把他加进 sudo 组。

Ubuntu 系统下：

bash
usermod -aG sudo 用户名


CentOS / Rocky Linux 系统下（搬瓦工默认常装这个）：

bash
usermod -aG wheel 用户名


加完之后让对方重新登录，执行 `sudo whoami` 显示 `root` 就生效了。

说实话，不是所有人都需要 sudo。如果只是让朋友跑个脚本、用 Python 跑点东西，普通账号够了。sudo 权限给了之后，对方一条 `sudo rm -rf /` 就能把你整台机器搞废，所以别随手给。

---

### 第三种场景：删除不再需要的账号

有人不用了，直接删干净：

bash
userdel -r 用户名


`-r` 参数会连同 `/home/用户名` 目录一起删掉。不加 `-r` 的话目录还在，占着空间。

---

### 第四种场景：给不同用户建独立的代理端口（多端口多密码）

这个场景略有不同——不是建 Linux 系统账号，而是在代理服务的配置文件里给每个人分一个独立端口和密码。

以常见的代理服务为例，配置文件里支持 `port_password` 格式，一个端口对应一个密码：

json
{
  "server": "0.0.0.0",
  "port_password": {
    "8381": "密码A",
    "8382": "密码B",
    "8383": "密码C"
  },
  "timeout": 300,
  "method": "aes-256-cfb"
}


每个朋友拿自己对应的端口和密码，互相不知道对方的配置。改完配置重启服务生效。

这样每个人用量可以独立追踪（用 iptables 统计各端口流量），万一某个账号有问题，直接从配置里删那一条，其他人不受影响。

## 配置多用户之后还要做什么？

建完账号之后有几件事值得顺手做一下。

**禁用 root 直接 SSH 登录**。平时用 root 操作太危险，而且暴力破解攻击基本都在试 root 账号。给每个人建好账号之后，编辑 `/etc/ssh/sshd_config`，把 `PermitRootLogin yes` 改成 `no`，重启 SSH 服务。以后要 root 权限就 sudo。

**换一个 SSH 端口**。搬瓦工本来就不用 22 端口，这点它做得比较好。如果你重装系统后改回 22 了，建议再换一个高位端口（比如 3 万多），能少很多无效扫描。

**给重要账号配密钥登录**。密码登录方便，但密钥更安全。本地生成一对密钥，把公钥复制到服务器 `~/.ssh/authorized_keys`，之后登录不用输密码，而且就算密码泄露也没人能登进去。

## 常见问题 FAQ

**Q：搬瓦工一台 VPS 能建多少个用户？**

Linux 系统对用户数量没有硬性限制，理论上几十个都可以。实际瓶颈是内存和带宽——人多了同时跑的进程多，内存不够就会变慢，流量也是大家分着用的。

**Q：多用户共用会不会互相影响？**

如果大家都是普通账号（没有 root/sudo），文件系统层面是隔离的，`/home/A` 里的东西 B 看不到也改不了。但网络带宽是共享的，某个人在大量下载，其他人的连接速度就会受影响。

**Q：配置了多端口代理之后，原来的账号还能用吗？**

切换到多端口配置文件之后，原来的单账号配置就失效了。改之前记录好原来的配置，新配置文件里把原来那个端口和密码也加进去就能延续。

**Q：搬瓦工 VPS 的 SSH 端口在哪里查？**

登录 KiwiVM 面板（后台），在 Main controls 页面直接能看到 SSH Port 那一行。不是 22，是一个几万的随机端口。

**Q：建了新用户之后，对方怎么改密码？**

用新账号 SSH 登录之后，直接执行 `passwd` 命令，按提示输入旧密码和新密码就可以改，不需要 root 权限。
